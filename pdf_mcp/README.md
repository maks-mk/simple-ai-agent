# PDFMcp

MCP-сервер для работы с PDF-файлами: создание PDF из Markdown, извлечение текста и изображений, поиск, редактирование, склейка, разбиение и поворот страниц.

## Возможности

| Инструмент | Описание |
|---|---|
| `pdf_create` | Создание PDF из Markdown-текста |
| `pdf_extract_text` | Извлечение текста постранично |
| `pdf_search` | Поиск текста с координатами и контекстом |
| `pdf_edit_text` | Замена или удаление существующего текста |
| `pdf_add_text` | Добавление нового текста на страницу |
| `pdf_merge` | Объединение нескольких PDF |
| `pdf_split` | Разбиение PDF по диапазонам страниц |
| `pdf_rotate_pages` | Поворот страниц на 90 / 180 / 270 |
| `pdf_extract_images` | Извлечение изображений в ZIP |
| `pdf_from_images` | Сборка PDF из списка изображений |

## Требования

- Python `>= 3.10`
- `uv` или `pip`

## Установка

### Через `uv`

```bash
uv run pdf_server.py
```

### Через `pip`

```bash
pip install "mcp[cli]" pymupdf reportlab markdown
python pdf_server.py
```

## Настройка MCP-клиента

### Claude Desktop

```json
{
  "mcpServers": {
    "pdfmcp": {
      "command": "uv",
      "args": [
        "run",
        "--with", "mcp[cli]",
        "--with", "pymupdf",
        "--with", "reportlab",
        "--with", "markdown",
        "/ABSOLUTE/PATH/TO/pdf_server.py"
      ]
    }
  }
}
```

### Claude Code

```bash
claude mcp add pdfmcp -- uv run --with "mcp[cli]" --with pymupdf --with reportlab --with markdown /ABSOLUTE/PATH/TO/pdf_server.py
```

### Другие MCP-клиенты

```json
{
  "mcpServers": {
    "pdfmcp": {
      "command": "python",
      "args": ["/absolute/path/to/pdf_server.py"],
      "transport": "stdio"
    }
  }
}
```

## Шрифты и кириллица

Сервер использует такую цепочку fallback:

1. локальные TTF в папке `pdf_mcp/fonts`
2. системные шрифты с поддержкой кириллицы
3. автоматическая загрузка Roboto в системную временную директорию

Для inline code и code blocks сервер сначала пытается использовать настоящий моноширинный шрифт с поддержкой кириллицы. Если такой шрифт не найден, код рендерится тем же Unicode-safe шрифтом, что и обычный текст, чтобы не терять кириллицу.

## Поддерживаемый Markdown в `pdf_create`

```markdown
# Заголовок 1
## Заголовок 2
### Заголовок 3
#### Заголовок 4

**жирный**, __жирный__
*курсив*
`инлайн код`
[ссылка](https://example.com)

---

- пункт списка
* пункт списка

> цитата

| Колонка 1 | Колонка 2 |
|-----------|-----------|
| ячейка    | ячейка    |

блок кода

![подпись](/путь/к/картинке.png)
```

Если `title` и первая строка `content` совпадают (`# Заголовок`), дубликат первого заголовка автоматически пропускается.

## Инструменты

### `pdf_create`

Создаёт PDF из Markdown.

| Параметр | Тип | Описание |
|---|---|---|
| `output_path` | `str` | Путь для сохранения PDF |
| `title` | `str` | Заголовок документа |
| `content` | `str` | Markdown-содержимое |

### `pdf_extract_text`

Извлекает текст постранично. Для больших файлов лучше читать документ чанками.

| Параметр | Тип | Описание |
|---|---|---|
| `pdf_path` | `str` | Путь к PDF |
| `page_start` | `int` | Первая страница, по умолчанию `1` |
| `page_end` | `int?` | Последняя страница включительно, по умолчанию все |
| `sort` | `bool` | Сортировать текст по координатам |

### `pdf_search`

Ищет текст и возвращает страницу, индекс вхождения, координаты и контекст.

| Параметр | Тип | Описание |
|---|---|---|
| `pdf_path` | `str` | Путь к PDF |
| `query` | `str` | Текст для поиска |
| `case_sensitive` | `bool` | Учитывать регистр |

### `pdf_edit_text`

Заменяет или удаляет существующий текст на выбранной странице. Для повторяющихся фрагментов сначала удобно вызвать `pdf_search`, а затем использовать `occurrence`.

| Параметр | Тип | Описание |
|---|---|---|
| `pdf_path` | `str` | Путь к PDF |
| `page_num` | `int` | Номер страницы, начиная с `1` |
| `old_text` | `str` | Текст для замены |
| `new_text` | `str` | Новый текст или пустая строка для удаления |
| `output_path` | `str?` | Путь для сохранения, по умолчанию `*_edited.pdf` |
| `occurrence` | `int` | `0` для всех вхождений, `1` для первого и т.д. |

### `pdf_add_text`

Добавляет новый текст на страницу.

| Параметр | Тип | Описание |
|---|---|---|
| `pdf_path` | `str` | Путь к PDF |
| `page_num` | `int` | Номер страницы, начиная с `1` |
| `text` | `str` | Текст для вставки |
| `x` | `float` | Координата X, по умолчанию `56.0` |
| `y` | `float?` | Координата Y; если не указана, текст добавляется после последнего блока |
| `fontsize` | `float` | Размер шрифта, по умолчанию `10.0` |
| `output_path` | `str?` | Путь для сохранения, по умолчанию `*_edited.pdf` |

### `pdf_merge`

Объединяет несколько PDF в один файл.

| Параметр | Тип | Описание |
|---|---|---|
| `pdf_paths` | `list[str]` | Список PDF в нужном порядке |
| `output_path` | `str` | Путь результата |

### `pdf_split`

Разбивает PDF на части по диапазонам страниц.

| Параметр | Тип | Описание |
|---|---|---|
| `pdf_path` | `str` | Путь к PDF |
| `ranges` | `list[str]` | Диапазоны, например `["1-3", "5"]` |
| `output_dir` | `str` | Папка для выходных файлов |

### `pdf_rotate_pages`

Поворачивает все страницы или выбранный список страниц.

| Параметр | Тип | Описание |
|---|---|---|
| `pdf_path` | `str` | Путь к PDF |
| `rotation` | `int` | Угол: `90`, `180`, `270` |
| `pages` | `list[int]?` | Номера страниц; если не указаны, поворачиваются все |

### `pdf_extract_images`

Извлекает изображения из PDF в ZIP-архив.

| Параметр | Тип | Описание |
|---|---|---|
| `pdf_path` | `str` | Путь к PDF |
| `output_path` | `str?` | Путь ZIP, по умолчанию рядом с исходным PDF |

### `pdf_from_images`

Собирает PDF из списка изображений.

| Параметр | Тип | Описание |
|---|---|---|
| `image_paths` | `list[str]` | Пути к изображениям |
| `output_path` | `str` | Путь для сохранения PDF |

## Ограничения

- Максимальный размер PDF: `100 MB`
- Поддерживаются только файлы с расширением `.pdf`
- `pdf_edit_text` использует redaction: исходный текст закрашивается, затем поверх вставляется новый
- Для корректной кириллицы нужен доступный Unicode-шрифт из цепочки fallback

## Устранение проблем

**Кириллица отображается квадратиками**  
Проверьте, что доступны локальные шрифты в `pdf_mcp/fonts`, системные шрифты с кириллицей или интернет для первой загрузки Roboto.

**Сервер не появляется в MCP-клиенте**  
Проверьте абсолютный путь к `pdf_server.py`, валидность JSON-конфига и перезапустите клиент.

**`uv` не найден**  
Установите `uv` или используйте запуск через `python`.

**Редактирование затрагивает не то вхождение**  
Сначала вызовите `pdf_search`, чтобы увидеть индексы вхождений, затем передайте нужный `occurrence`.
